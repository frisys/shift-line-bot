# シフト管理システム 仕様書

最終更新: 2026-04-04

---

## 1. システム概要

スタッフシフト管理システム。
LINE Bot を通じてスタッフがシフト希望を提出し、店長が Web ダッシュボードで確認・管理する。

### 対象ユーザー

| ユーザー種別 | 利用方法 | 主な機能 |
|-------------|---------|---------|
| スタッフ | LINE Bot + 内部ブラウザ | シフト希望提出・確認、店舗登録（複数店舗可） |
| 店長 | Web ダッシュボード（`/dashboard`） | シフト希望確認、スタッフ管理、必要人数設定 |

### ユーザーと店舗の関係

- **1ユーザー : N店舗** の関係
- 1人のスタッフが複数の店舗に所属可能
- 店舗ごとに役割・勤務制約を個別設定

## 2. LINE Bot 機能

### 2.1 友達追加時の処理

1. ユーザーが Bot を友達追加
2. `profiles` テーブルに LINE 表示名で upsert
3. リッチメニューを新規作成してユーザーに紐づけ（`createAndSetRichMenu`）
4. 挨拶メッセージを送信し、店舗コード入力を促す

### 2.2 店舗登録

1. スタッフが店舗コード（4〜10文字の英数字）を送信
2. `stores` テーブルから該当店舗を検索（大文字化して照合）
3. 登録確認 Flex メッセージを表示
   - 「登録する」（`displayText: '登録する'`）→ postback: `register_store`
   - 「やめる」（`displayText: 'やめる'`）→ postback: `cancel_register_store`
4. 「登録する」タップ直後、チャットにユーザー発言として「登録する」が表示される
5. `user_stores` テーブルに `staff` ロールで upsert
6. 登録完了後、切り替え確認 Flex メッセージを表示
   - 「切り替えて」（`displayText: '切り替えて'`）→ postback: `confirm_switch_store`
   - 「今のままで」（`displayText: '今のままで'`）→ postback: `keep_current_store`
7. リッチメニューをユーザーに再適用（`createAndSetRichMenu`）

**店舗コード形式:** 英数字 4〜10 文字（例: `ABC123`, `STORE001`）

### 2.3 店舗切り替え

複数店舗に登録しているスタッフが、操作対象の店舗を切り替える機能。

1. リッチメニューから「店舗切り替え」を選択
2. 登録済み店舗一覧を Flex カルーセルで表示
3. 店舗を選択すると `profiles.active_store_id` を更新
4. 切り替え完了メッセージを表示

### 2.4 シフト希望提出フロー（LINE Bot 側）

```
[シフト希望提出] ボタン
    ↓
月選択メッセージ（今月 / 来月）
  ※ URI アクションで /shift-select?year=...&month=...&userId=...&storeId=... を開く
    ↓
内部ブラウザで /shift-select 画面を表示
```

### 2.5 シフト希望確認

1. リッチメニューから「シフト希望を確認」を選択
2. 今月・翌月のシフト希望を Flex カレンダーで表示
3. 確定済みの月は「確定」ラベルを表示

### 2.6 リッチメニュー

| エリア | アクション |
|--------|-----------|
| 左上 | シフト希望を提出（postback: `submit_shift`） |
| 右上 | シフト希望を確認（postback: `view_preferences`） |
| 左下 | 店舗を切り替える（postback: `change_store`） |
| 右下 | ヘルプ（URI リンク） |

## 3. シフト希望入力画面（`/shift-select`）

LINE Bot の内部ブラウザで開くスタッフ向けの Web 画面。

### アクセス URL

```
/shift-select?year=YYYY&month=M&userId=<lineUserId>&storeId=<storeId>
```

### 機能一覧

| 機能 | 説明 |
|------|------|
| カレンダー表示 | 対象月の日付をグリッド表示（日曜赤・土曜青・平日グレー） |
| ステータス選択 | 日付セルをタップするたびに `−` → `◯` → `△` → `×` → `−` とサイクル |
| 時間帯選択 | ◯ または △ の日のみ時間帯ドロップダウンを表示 |
| 曜日一括設定 | カレンダー曜日ヘッダー（日〜土）をタップするたびに `◯` → `△` → `×` → 解除 のサイクルで全日程を一括変更。全日程が同一ステータスの場合はそのステータスを次にサイクル、バラバラの場合は `◯` から開始。アクティブ状態はボタン色で表示 |
| 過去シフト確認 | ヘッダーのプルダウン（過去 12 ヶ月）で月を選び「確認する」を押すと、読み取り専用のカレンダーモーダルで表示 |
| リセット | 「リセット」ボタンで全選択をクリア |
| 保存 | 「シフト希望を保存」ボタンで `/api/shift-preferences` に POST |
| 凡例 | `◯` 出勤可 / `△` 微妙 / `×` 休み / `−` 希望なし |

### ステータス定義

| 記号 | 値 | 色 | 意味 |
|------|----|----|------|
| ◯ | `ok` | 緑 | 出勤可 |
| △ | `maybe` | 黄 | 微妙 |
| × | `no` | 赤 | 休み |
| − | `null` | グレー | 希望なし（未選択） |

### 時間帯

`早番` / `日勤` / `遅番` / `夜勤` / `フル`

## 4. 管理画面（店長向け）

### 4.1 ログイン画面（`/login`）

- メールアドレス + パスワードによる Supabase Auth 認証
- Enter キーでログイン可能
- エラー時はカード内にエラーメッセージを表示
- ログイン成功後 `/dashboard` へリダイレクト

### 4.2 メイン画面（`/dashboard`）

ログイン後のメイン画面。タブ切り替えで 3 つのセクションを表示。

**ヘッダー（スティッキー）**
- アプリタイトル
- 複数店舗の場合は店舗セレクタ（プルダウン）
- ログインユーザーのメールアドレス
- ログアウトボタン

**タブ構成**

| タブ | 内容 |
|------|------|
| シフト希望 | 週単位のスタッフ別シフト希望一覧 |
| スタッフ | 店舗スタッフ一覧・編集 |
| 店舗設定 | 店舗情報・曜日別必要人数 |

### 4.3 シフト希望タブ

**表示形式:** 週単位のカレンダー（スタッフ × 曜日のテーブル）

| スタッフ | 日 | 月 | 火 | 水 | 木 | 金 | 土 |
|---------|---|---|---|---|---|---|---|
| 田中 | ◯ | △ | × | ◯ | ◯ | - | ◯ |
| 佐藤 | × | ◯ | ◯ | △ | ◯ | ◯ | - |

**凡例:** ◯（緑）出勤可 / △（黄）微妙 / ×（赤）休み / -（グレー）未提出

**機能:**
- 前週 / 次週ナビゲーション
- 今週に戻るボタン
- 時間帯表示
- 備考のホバーツールチップ

**データ取得範囲:** 前月 1 日〜3 ヶ月先の末日（動的計算）

### 4.4 スタッフタブ

**表示項目:** 名前 / 役割 / 最大連勤日数 / 週最大勤務日数 / 苦手曜日

**編集モーダルで変更可能な項目:**
- 名前
- 役割（店長 / スタッフ / 管理者）
- 最大連勤日数
- 週最大勤務日数

### 4.5 店舗設定タブ

**表示項目:** 店舗名 / 店舗コード / 曜日別必要人数（日〜土）

**編集機能:**
- 曜日別必要人数の変更（0〜10 人）
- 変更差分を確認ダイアログで確認してから保存

## 5. API 仕様

### LINE Webhook

**エンドポイント:** `POST /api/line/webhook`

**認証:** HMAC-SHA256 署名検証（`x-line-signature` ヘッダー）

**処理イベント:**

| イベント | 処理 |
|---------|------|
| follow | プロフィール登録、リッチメニュー作成・設定、挨拶メッセージ送信 |
| message (text) | 店舗コード入力 → 登録確認フロー |
| postback | 下記 postback アクション参照 |

**postback アクション:**

| action | 処理 |
|--------|------|
| `submit_shift` | 月選択メニュー表示 |
| `view_preferences` | シフト希望確認（Flex カレンダー） |
| `change_store` | 登録済み店舗一覧表示 |
| `switch_store` | 店舗切り替え実行 |
| `register_store` | 店舗登録実行 |
| `cancel_register_store` | 店舗登録キャンセル |
| `confirm_switch_store` | 登録後の店舗切り替え実行 |
| `keep_current_store` | 登録後の店舗切り替えスキップ |
| `noop` | 何もしない（カレンダー空白セル用） |

### シフト希望 API

**エンドポイント:** `GET /api/shift-preferences`

| パラメータ | 説明 |
|-----------|------|
| userId | LINE ユーザー ID |
| storeId | 店舗 ID |
| year | 年 |
| month | 月 |

**エンドポイント:** `POST /api/shift-preferences`

```json
{
  "preferences": [
    {
      "user_id": "Uxxxx",
      "store_id": "uuid",
      "shift_date": "2026-05-01",
      "status": "ok",
      "time_slot": "日勤"
    }
  ]
}
```

### 店舗情報 API

**エンドポイント:** `GET /api/stores/[id]`

## 6. データベース設計

### profiles テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | TEXT | 表示名 |
| line_user_id | TEXT | LINE ユーザー ID（ユニーク） |
| phone | TEXT | 電話番号（ユニーク） |
| active_store_id | UUID | 現在選択中の店舗 ID（NULL 可） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### stores テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | TEXT | 店舗名 |
| owner_user_id | UUID | オーナーのユーザー ID |
| store_code | TEXT | 店舗コード（4〜10 文字、ユニーク） |
| required_staff | JSONB | 曜日別必要人数 `{mon: 3, tue: 4, ...}` |
| address | TEXT | 住所（任意） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### user_stores テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| user_id | TEXT | LINE ユーザー ID |
| store_id | UUID | 店舗 ID |
| role | TEXT | 役割（manager / staff / admin） |
| joined_at | TIMESTAMP | 登録日時 |
| max_consecutive_days | INT | 最大連勤日数 |
| max_weekly_days | INT | 週最大勤務日数 |
| unavailable_days | TEXT[] | 出勤不可曜日 |
| preferred_time_slots | TEXT[] | 希望時間帯 |

**主キー（ユニーク制約）:** `(user_id, store_id)`

### shift_preferences テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| user_id | TEXT | LINE ユーザー ID |
| store_id | UUID | 店舗 ID |
| shift_date | DATE | シフト日（YYYY-MM-DD） |
| status | TEXT | 希望状態（ok / maybe / no） |
| time_slot | TEXT | 時間帯（早番 / 日勤 / 遅番 / 夜勤 / フル） |
| note | TEXT | 備考 |
| submitted_at | TIMESTAMP | 提出日時 |
| updated_at | TIMESTAMP | 更新日時 |

**ユニーク制約:** `(user_id, store_id, shift_date)`

### shift_confirmations テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| store_id | UUID | 店舗 ID |
| year_month | TEXT | 対象年月（"2026-03" 形式） |
| is_confirmed | BOOLEAN | 確定済みか（デフォルト: false） |
| created_at | TIMESTAMP | 募集開始日時 |
| confirmed_at | TIMESTAMP | 確定日時（NULL なら未確定） |

**ユニーク制約:** `(store_id, year_month)`

## 9. 管理画面 動作確認手順

### 前提条件

- `npm run dev` でローカルサーバーが起動していること（ポート 3000）
- Supabase に店長ユーザーが作成済みであること
- その店長ユーザーの `owner_user_id` に紐づいた店舗が `stores` テーブルに存在すること

### 9.1 ログイン確認

1. ブラウザで `http://localhost:3000/login` を開く
2. 以下を確認する
   - 「シフト管理システム」「店長向けポータル」の表示
   - メールアドレス・パスワードの入力フォームが表示される
3. 誤ったパスワードを入力して「ログイン」を押す
   - 「メールアドレスまたはパスワードが正しくありません」が表示されること
4. 正しいメールアドレス・パスワードを入力して「ログイン」を押す
   - `/dashboard` へ遷移すること
5. Enter キーでもログインできることを確認する

### 9.2 ダッシュボード初期表示確認

1. ログイン後、`/dashboard` が表示されることを確認する
2. ヘッダーに以下が表示されることを確認する
   - 「📅 シフト管理」
   - 店舗名（複数店舗の場合はプルダウン）
   - ログイン中のメールアドレス
   - 「ログアウト」ボタン
3. 「シフト希望」「スタッフ」「店舗設定」の 3 つのタブが表示されることを確認する

### 9.3 シフト希望タブ確認

1. 「シフト希望」タブを選択する
2. スタッフ × 曜日のテーブルが表示されることを確認する
3. 「← 前週」「次週 →」「今週に戻る」ボタンで週を移動できることを確認する
4. スタッフが希望を提出している場合、◯ / △ / × と時間帯が表示されることを確認する
   - データが現在日付周辺（前月〜3 ヶ月先）で取得されることを確認する
5. 「今週に戻る」ボタンは今週表示中は非活性になることを確認する

### 9.4 スタッフタブ確認

1. 「スタッフ」タブを選択する
2. 登録済みスタッフの一覧（名前・役割・最大連勤・週最大・苦手曜日）が表示されることを確認する
3. 「編集」ボタンを押してモーダルが開くことを確認する
4. 名前・役割・連勤日数・週最大日数を変更して保存できることを確認する
5. スタッフがいない場合は「スタッフが登録されていません」と表示されることを確認する

### 9.5 店舗設定タブ確認

1. 「店舗設定」タブを選択する
2. 店舗名・店舗コード・曜日別必要人数が表示されることを確認する
3. 「編集」ボタンを押して各曜日の人数を変更できることを確認する
4. 「保存」を押すと変更差分を示す確認ダイアログが表示されることを確認する
5. 確認後「保存する」でデータが更新されることを確認する
6. 「キャンセル」で元の値に戻ることを確認する

### 9.6 複数店舗切り替え確認（複数店舗がある場合）

1. ヘッダーの店舗プルダウンで別の店舗を選択する
2. 各タブの表示データが選択した店舗のものに切り替わることを確認する
3. ページリロード後も前回選択した店舗が維持されることを確認する（`localStorage` 保持）

### 9.7 ログアウト確認

1. ヘッダーの「ログアウト」ボタンを押す
2. `/login` へ遷移することを確認する
3. ブラウザバックしても `/dashboard` にアクセスできない（`/login` にリダイレクトされる）ことを確認する
