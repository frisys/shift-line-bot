# シフト管理システム 仕様書

## 1. システム概要

スタッフシフト管理システム。
LINE Botを通じてスタッフがシフト希望を提出し、店長がWebダッシュボードで確認・管理する。

### 対象ユーザー

| ユーザー種別 | 利用方法 | 主な機能 |
|-------------|---------|---------|
| スタッフ | LINE Bot | シフト希望提出、店舗登録（複数店舗可） |
| 店長 | Webダッシュボード | スタッフ管理、シフト希望確認、必要人数設定 |

### ユーザーと店舗の関係

- **1ユーザー : N店舗** の関係
- 1人のスタッフが複数の店舗に所属可能
- 店舗ごとに役割・勤務制約を設定可能

## 2. LINE Bot機能

### 2.1 友達追加時の処理

1. ユーザーがBotを友達追加
2. `profiles`テーブルにLINE表示名で登録
3. リッチメニューを設定
4. 挨拶メッセージを送信し、店舗コード入力を促す

**挨拶メッセージ例:**
```
○○さん！お疲れ様です！
シフト管理君を追加してくれてありがとうございます！

まずは働く店舗の登録をしましょう。

リッチメニューから店舗登録を選択し、
店長からもらった店舗コードを入力してください。
例: ABC123

※複数の店舗で働く場合は、その都度リッチメニューから登録してください。
```

### 2.2 店舗登録

1. スタッフが店舗コード（4〜10文字の英数字）を送信
2. `stores`テーブルから該当店舗を検索
3. 見つかった場合、登録確認メッセージを表示（「登録する」「やめる」）
4. 「登録する」を選択 → `user_stores`テーブルに`staff`ロールで登録
5. 登録完了後、切り替え確認メッセージを表示（「切り替えて」「今のままで」）
6. 「切り替えて」を選択 → `profiles.active_store_id`を更新

**店舗コード形式:** 英数字4〜10文字（例: `ABC123`, `STORE001`）

### 2.3 店舗切り替え

複数店舗に登録しているスタッフが、操作対象の店舗を切り替える機能。

1. リッチメニューから「店舗切り替え」を選択
2. 登録済み店舗一覧をFlexカルーセルで表示
3. 店舗を選択すると`profiles.active_store_id`を更新
4. 切り替え完了メッセージを表示

**登録店舗がない場合:**
「現在登録されている店舗はありません。新しい店舗コードを入力してください！」

### 2.4 シフト希望提出フロー

```
[シフト希望提出] ボタン
    ↓
月選択
  ○月（今月） / ○月（来月）
    ↓
内部ブラウザでシフト管理画面（カレンダー形式）を表示
    ↓
日付選択
  ※日曜=赤、土曜=青、平日=緑
    ↓
希望状態と時間帯選択用のモーダル表示
    ↓
希望状態選択
  ◯ 出勤可 (ok)
  △ 微妙 (maybe)
  × 休み (no)
    ↓
時間帯選択
  早番 / 日勤 / 遅番 / 夜勤 / フル / 休み希望
    ↓
`shift_preferences`テーブルに保存
    ↓
完了メッセージ送信
    ↓
内部ブラウザを閉じる
```

### 2.5 シフト希望確認

登録したシフト希望をカレンダー形式で確認する機能。

1. リッチメニューから「シフト希望を確認」を選択
2. 今月のシフト希望をFlexカレンダーで表示
3. 翌月分の希望がある場合はカルーセル形式で今月・翌月を表示
4. 確定済みの月は「確定」ラベルを表示

### 2.6 リッチメニュー

| エリア | アクション |
|--------|-----------|
| 左上 | シフト希望を提出 |
| 右上 | シフト希望を確認 |
| 左下 | 店舗を切り替える |
| 右下 | ヘルプ（外部リンク） |

## 3. Webダッシュボード機能

### 3.1 認証

- Supabase Authによるメール/パスワード認証
- ログイン後、`owner_user_id`が一致する店舗のみ表示

### 3.2 店舗情報表示・編集

**表示項目:**
- 店舗名
- 店舗コード（スタッフに共有する用）
- 曜日別必要人数

**編集機能:**
- 曜日別必要人数の変更（0〜10人）
- 変更時は確認ダイアログを表示

### 3.3 スタッフ一覧

**表示項目:**
- 名前
- 役割（店長/スタッフ/管理者）
- 最大連勤日数
- 週最大勤務日数

**編集機能:**
- 名前変更
- 役割変更
- 勤務制約変更（最大連勤日数、週最大日数）

### 3.4 シフト希望一覧

**表示形式:** 週単位のカレンダー形式

| スタッフ | 日 | 月 | 火 | 水 | 木 | 金 | 土 |
|---------|---|---|---|---|---|---|---|
| 田中 | ◯ | △ | × | ◯ | ◯ | - | ◯ |
| 佐藤 | × | ◯ | ◯ | △ | ◯ | ◯ | - |

**凡例:**
- ◯ (緑): 出勤可
- △ (黄): 微妙
- × (赤): 休み
- -: 未提出

**機能:**
- 前週/次週ナビゲーション
- 今週に戻るボタン
- 時間帯のツールチップ表示
- 備考のホバー表示

### 3.5 シフト募集・確定管理（未実装）

> ⚠️ **TODO: 実装が必要**

店長がシフト希望の募集開始・確定を行う機能。

**募集開始:**
- 対象月を選択して「シフト希望を募集」ボタンを押下
- `shift_confirmations`テーブルにレコード作成（is_confirmed = false）
- スタッフがLINE Botでシフト希望を提出可能になる

**シフト確定:**
- 対象月の「シフトを確定」ボタンを押下
- `shift_confirmations`テーブルを更新（is_confirmed = true, confirmed_at = now()）
- スタッフがLINE Botでシフト確認時に「確定」と表示される

**確定解除:**
- 確定済みの月の「確定を解除」ボタンを押下
- `shift_confirmations`テーブルを更新（is_confirmed = false, confirmed_at = NULL）

## 4. データベース設計

### 4.1 profiles テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | TEXT | 表示名 |
| line_user_id | TEXT | LINE ユーザーID（ユニーク） |
| phone | TEXT | 電話番号（ユニーク） |
| active_store_id | UUID | 現在選択中の店舗ID（外部キー、NULL可） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### 4.2 stores テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | TEXT | 店舗名 |
| owner_user_id | UUID | オーナーのユーザーID |
| store_code | TEXT | 店舗コード（4〜10文字） |
| required_staff | JSONB | 曜日別必要人数 `{mon: 3, tue: 4, ...}` |
| address | TEXT | 住所（任意） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### 4.3 user_stores テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| user_id | TEXT | LINE ユーザーID |
| store_id | UUID | 店舗ID |
| role | TEXT | 役割（manager/staff/admin） |
| joined_at | timestamp | 登録日時 |
| max_consecutive_days | INT | 最大連勤日数 |
| max_weekly_days | INT | 週最大勤務日数 |
| unavailable_days | TEXT[] | 出勤不可曜日 |
| preferred_time_slots | TEXT[] | 希望時間帯 |

**主キー:** `(user_id, store_id)`

### 4.4 shift_preferences テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| user_id | TEXT | LINE ユーザーID |
| store_id | UUID | 店舗ID |
| shift_date | DATE | シフト日（YYYY-MM-DD） |
| status | TEXT | 希望状態（ok/maybe/no） |
| time_slot | TEXT | 時間帯（早番/日勤/遅番/夜勤/フル） |
| note | TEXT | 備考 |
| submitted_at | TIMESTAMP | 提出日時 |
| updated_at | TIMESTAMP | 更新日時 |

**ユニーク制約:** `(user_id, store_id, shift_date)`

### 4.5 shift_confirmations テーブル

シフトの募集・確定状態を管理するテーブル。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| store_id | UUID | 店舗ID（外部キー） |
| year_month | TEXT | 対象年月（"2026-03" 形式） |
| is_confirmed | BOOLEAN | 確定済みか（デフォルト: false） |
| created_at | TIMESTAMP | 募集開始日時 |
| confirmed_at | TIMESTAMP | 確定日時（NULLなら未確定） |

**ユニーク制約:** `(store_id, year_month)`

**ライフサイクル:**
1. 店長がダッシュボードからシフト希望を募集 → レコード作成（is_confirmed = false）
2. スタッフがLINE Botでシフト希望を提出
3. 店長がダッシュボードで確定 → is_confirmed = true, confirmed_at = now()
4. （必要に応じて確定解除 → is_confirmed = false, confirmed_at = NULL）

**運用:**
- 古いレコードは年次で削除

## 5. API仕様

### LINE Webhook

**エンドポイント:** `POST /api/line/webhook`

**認証:** HMAC-SHA256署名検証（`x-line-signature`ヘッダー）

**処理イベント:**

| イベント | 処理 |
|---------|------|
| follow | プロフィール登録、リッチメニュー設定、挨拶メッセージ送信 |
| message (text) | 店舗コード入力 → 登録確認フロー |
| postback | 下記postbackアクション参照 |

**postbackアクション:**

| action | 処理 |
|--------|------|
| submit_shift | シフト希望提出（月選択メニュー表示） |
| view_preferences | シフト希望確認（カレンダー表示） |
| change_store | 店舗切り替え（登録店舗一覧表示） |
| switch_store | 店舗切り替え実行 |
| register_store | 店舗登録実行 |
| cancel_register_store | 店舗登録キャンセル |
| confirm_switch_store | 登録後の店舗切り替え実行 |
| keep_current_store | 登録後の店舗切り替えスキップ |
| select_month | 月選択 |
| select_date | 日付選択 |
| select_status | 希望状態選択 |
| select_time_slot | 時間帯選択・保存 |

## 7. 画面遷移

### LINE Bot

```友達追加
友達追加
    ↓
挨拶メッセージ
    ↓
リッチメニューから操作
  ├─ シフト希望提出 → 月選択 → カレンダー表示（内部ブラウザ） → 日付選択 → 状態選択 → 時間帯選択 → (必要分繰り返し) → 完了 or キャンセル → 完了
  ├─ シフト希望確認 → シフト希望(or確定)をFlexカレンダーで表示
  ├─ 店舗切り替え → 登録済み店舗一覧表示 → 店舗選択 → 切り替え完了
  └─ 店舗登録 → 店舗コード入力 → 登録確認 → 登録完了 → 切り替え確認
```

```シフト希望提出
シフト希望提出
    ↓
店舗選択
    ↓
月選択
    ↓
カレンダー表示（内部ブラウザ）
    ↓
カレンダーから日付選択
    ↓
状態と時間帯選択モーダル表示
    ↓
希望状態と時間帯(複数選択)選択
    ↓
登録
    ↓
モーダル閉じる
    ↓
（必要な分繰り返す）
    ↓
登録押下
    ↓
`shift_preference`に登録
    ↓
ブラウザ閉じる
    ↓
「登録が完了しました」
```

```シフト希望確認
シフト希望確認
    ↓
登録店舗の今月・翌月のシフト希望を取得
    ↓
shift_confirmationsテーブルから確定状態を取得
    ↓
Flexカレンダー形式で表示（確定月は「確定」ラベル付き）
    ↓
翌月分の希望がある場合はカルーセル形式で表示
```

```店舗切り替え
店舗切り替え
    ↓
登録済み店舗を取得
    ↓ 店舗あり
Flexカルーセルで店舗一覧表示
    ↓
店舗選択（「この店舗に切り替え」ボタン）
    ↓
profiles.active_store_id を更新
    ↓
「○○に切り替えました！」
```

```店舗登録
店舗登録
    ↓
店舗コード入力促し
    ↓
店舗コード入力
    ↓
登録確認（「登録する」「やめる」）
    ↓ 「登録する」選択
店舗登録完了
    ↓
切り替え確認（「切り替えて」「今のままで」）
    ↓ 「切り替えて」選択
active_store_id更新・切り替え完了
```

### Webダッシュボード

```
/login（ログイン）
    ↓
/dashboard（ダッシュボード）
  ├─ 店舗選択（複数店舗の場合）
  ├─ 店舗情報セクション
  │     └─ 必要人数編集モーダル
  ├─ スタッフ一覧セクション
  │     └─ スタッフ編集モーダル
  └─ シフト希望一覧セクション
        └─ 週ナビゲーション
```
