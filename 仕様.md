# シフト管理システム 仕様書

## 1. システム概要

バスケットボールコート（Skyhigh）のスタッフシフト管理システム。
LINE Botを通じてスタッフがシフト希望を提出し、店長がWebダッシュボードで確認・管理する。

### 対象ユーザー

| ユーザー種別 | 利用方法 | 主な機能 |
|-------------|---------|---------|
| スタッフ | LINE Bot | シフト希望提出、店舗登録（複数店舗可） |
| 店長 | Webダッシュボード | スタッフ管理、シフト希望確認、必要人数設定 |

### ユーザーと店舗の関係

- **1ユーザー : N店舗** の関係
- 1人のスタッフが複数の店舗に所属可能
- 店舗ごとに役割・勤務制約を設定可能

## 2. LINE Bot機能

### 2.1 友達追加時の処理

1. ユーザーがBotを友達追加
2. `profiles`テーブルにLINE表示名で登録
3. リッチメニューを設定
4. 挨拶メッセージを送信し、店舗コード入力を促す

**挨拶メッセージ例:**
```
こんにちは、○○さん！
シフト管理Botへようこそ！

まずは働く店舗の登録をしましょう。

店長からもらった店舗コードを入力してください。
例: ABC123

※複数の店舗で働く場合は、それぞれの店舗コードを順番に入力してください。
```

### 2.2 店舗登録

1. スタッフが店舗コード（4〜10文字の英数字）を送信
2. `stores`テーブルから該当店舗を検索
3. 見つかった場合、`user_stores`テーブルに`staff`ロールで登録
4. 登録完了メッセージを送信

**店舗コード形式:** 英数字4〜10文字（例: `ABC123`, `STORE001`）

### 2.3 シフト希望提出フロー

```
[シフト希望提出] ボタン
    ↓
月選択
  ○月（今月） / ○月（来月）
    ↓
日付選択（カレンダー形式）
  ※選択した月の全日付を週ごとに表示
  ※日曜=赤、土曜=青、平日=緑
    ↓
希望状態選択
  ◯ 出勤可 (ok)
  △ 微妙 (maybe)
  × 休み (no)
    ↓
時間帯選択
  早番 / 日勤 / 遅番 / 夜勤 / フル / 休み希望
    ↓
`shift_preferences`テーブルに保存
    ↓
完了メッセージ送信
```

### 2.4 リッチメニュー

| エリア | アクション |
|--------|-----------|
| 左上 | シフト希望を提出 |
| 右上 | シフト希望を確認 |
| 左下 | 店舗を切り替える |
| 右下 | ヘルプ（外部リンク） |

## 3. Webダッシュボード機能

### 3.1 認証

- Supabase Authによるメール/パスワード認証
- ログイン後、`owner_user_id`が一致する店舗のみ表示

### 3.2 店舗情報表示・編集

**表示項目:**
- 店舗名
- 店舗コード（スタッフに共有する用）
- 曜日別必要人数

**編集機能:**
- 曜日別必要人数の変更（0〜10人）
- 変更時は確認ダイアログを表示

### 3.3 スタッフ一覧

**表示項目:**
- 名前
- 役割（店長/スタッフ/管理者）
- 最大連勤日数
- 週最大勤務日数

**編集機能:**
- 名前変更
- 役割変更
- 勤務制約変更（最大連勤日数、週最大日数）

### 3.4 シフト希望一覧

**表示形式:** 週単位のカレンダー形式

| スタッフ | 日 | 月 | 火 | 水 | 木 | 金 | 土 |
|---------|---|---|---|---|---|---|---|
| 田中 | ◯ | △ | × | ◯ | ◯ | - | ◯ |
| 佐藤 | × | ◯ | ◯ | △ | ◯ | ◯ | - |

**凡例:**
- ◯ (緑): 出勤可
- △ (黄): 微妙
- × (赤): 休み
- -: 未提出

**機能:**
- 前週/次週ナビゲーション
- 今週に戻るボタン
- 時間帯のツールチップ表示
- 備考のホバー表示

## 4. データベース設計

### 4.1 profiles テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| line_user_id | TEXT | LINE ユーザーID（ユニーク） |
| name | TEXT | 表示名 |

### 4.2 stores テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | TEXT | 店舗名 |
| owner_user_id | UUID | オーナーのユーザーID |
| store_code | TEXT | 店舗コード（4〜10文字） |
| required_staff | JSONB | 曜日別必要人数 `{mon: 3, tue: 4, ...}` |
| address | TEXT | 住所（任意） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### 4.3 user_stores テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| user_id | TEXT | LINE ユーザーID |
| store_id | UUID | 店舗ID |
| role | TEXT | 役割（manager/staff/admin） |
| line_user_id | TEXT | LINE ユーザーID |
| max_consecutive_days | INT | 最大連勤日数 |
| max_weekly_days | INT | 週最大勤務日数 |
| unavailable_days | TEXT[] | 出勤不可曜日 |
| preferred_time_slots | TEXT[] | 希望時間帯 |

**主キー:** `(user_id, store_id)`

### 4.4 shift_preferences テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| user_id | TEXT | LINE ユーザーID |
| store_id | UUID | 店舗ID |
| shift_date | DATE | シフト日（YYYY-MM-DD） |
| status | TEXT | 希望状態（ok/maybe/no） |
| time_slot | TEXT | 時間帯（早番/日勤/遅番/夜勤/フル） |
| note | TEXT | 備考 |
| submitted_at | TIMESTAMP | 提出日時 |
| updated_at | TIMESTAMP | 更新日時 |

**ユニーク制約:** `(user_id, store_id, shift_date)`

## 5. API仕様

### LINE Webhook

**エンドポイント:** `POST /api/line/webhook`

**認証:** HMAC-SHA256署名検証（`x-line-signature`ヘッダー）

**処理イベント:**

| イベント | 処理 |
|---------|------|
| follow | プロフィール登録、リッチメニュー設定 |
| message (text) | 店舗コード入力 or シフトメニュー表示 |
| postback | シフト希望提出フロー |

## 7. 画面遷移

### LINE Bot

```
友達追加
    ↓
挨拶メッセージ（店舗コード入力促し）
    ↓
店舗コード入力 → 登録完了（複数店舗登録可）
    ↓
リッチメニューから操作
  ├─ シフト希望提出 → 月選択 → 日付選択 → 状態選択 → 時間帯選択 → 完了
  ├─ シフト希望確認 → (未実装)
  └─ 店舗切り替え → 登録店舗一覧 → 選択
```

### Webダッシュボード

```
/login（ログイン）
    ↓
/dashboard（ダッシュボード）
  ├─ 店舗選択（複数店舗の場合）
  ├─ 店舗情報セクション
  │     └─ 必要人数編集モーダル
  ├─ スタッフ一覧セクション
  │     └─ スタッフ編集モーダル
  └─ シフト希望一覧セクション
        └─ 週ナビゲーション
```
